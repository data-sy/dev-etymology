# DevEtym 앱 아이콘 디자인 요청

## 너의 역할

너는 **10년차 시니어 브랜드 디자이너**다.
Stripe, Linear, Arc, Raycast 같은 제품의 아이덴티티를 만들어 본 감각을 가지고 있고,
iOS 앱 아이콘이 홈스크린이라는 지옥 같은 경쟁 환경에서 살아남아야 한다는 걸 안다.

나는 클라이언트다. 나는 내 앱의 맥락을 너보다 모른다.
너는 단순히 내 지시를 실행하는 도구가 아니라, 전문가로서 **네 판단으로 후보를 제시**해야 한다.

## 프로젝트 브리프

**앱 이름:** DevEtym
**한 줄 설명:** 개발 용어의 어원(etymology)을 파헤치는 iOS 사전 앱
**단계:** MVP
**타겟 사용자:** 개발자 (주 타겟: 한국어 사용자)

아이콘이 해야 하는 유일한 일:
**홈스크린에 수십 개 아이콘이 있는 상황에서, 0.5초 안에 "이건 뭔가 사전/레퍼런스 성격의 앱"이라고 인식되게 할 것.**

심볼적 차별화, 브랜드 철학, 메타포의 깊이 — 이런 건 지금 단계에서 사치다.
MVP에서 아이콘의 성공 기준은 오직 **카테고리 인식**이다.

## 작업 순서

### 1단계: 프로젝트 코드 분석

먼저 이 프로젝트를 훑어서 다음을 파악해줘:

**색상 팔레트 추출:**
- `Assets.xcassets/Colors/`, `AccentColor.colorset/`
- `Color+Extension.swift`, `Theme.swift`, `AppColor.swift` 같은 색상 정의 파일
- 주요 View 파일의 인라인 색 사용
- 추출 결과: primary / accent / background / surface (라이트·다크 모드 분리)

**앱 무드 파악:**
- `spec.md`, `CLAUDE.md`, `README.md`, 주요 View를 훑어서
- 폰트 성향, 레이아웃 밀도, 전반 톤을 한 줄로 요약

색상이 명확히 정의되어 있지 않으면 추측하지 말고 나에게 물어봐.

### 2단계: 후보 방향 자율 제안

**이 단계가 가장 중요하다.**

위에서 파악한 팔레트·무드·브리프를 종합해서, **시니어 디자이너로서 네가 판단하는** 5가지 아이콘 방향을 제시해라.

나는 의도적으로 방향 예시를 주지 않는다. "책을 그려라", "레터마크로 가라" 같은 가이드는 없다. 네가 전문가로서 판단할 부분이다.

다만 판단의 제약 조건은 다음과 같다:

- **제1원칙:** 0.5초 카테고리 인식. "사전/레퍼런스/학습" 계열로 읽혀야 함
- 28×28px (iOS 알림센터 사이즈)에서도 실루엣이 살아남을 것
- 앱이 사용하는 실제 색상 팔레트 안에서 구성할 것 (팔레트 확장 시 근거 제시)
- 5개 후보는 서로 뚜렷하게 다른 접근이어야 함 (비슷한 방향 5개 X)
- 각 후보는 **왜 이 방향을 선택했는지**의 논리를 가져야 함

네가 대안으로 "이런 접근이 MVP에 더 맞다"고 판단하면 그 판단을 관철해도 된다. 예를 들어 5개 중 2개는 카테고리 명확성 쪽, 2개는 차별화 쪽, 1개는 과감한 대안 — 이런 식으로 밸런스를 짜는 것도 네 판단이다.

각 후보마다 다음을 명시:
- **컨셉 이름** (짧게)
- **핵심 시각 요소** (1~2문장)
- **왜 이게 이 브리프에 맞는가** (2~3문장, 설득)
- **리스크 / 트레이드오프** (솔직하게)
- **28px에서 살아남을 요소** (실루엣 앵커)

### 3단계: HTML 결과물 생성

파일명: `devetym_icon_candidates.html`
위치: 프로젝트 루트 또는 `design/` 폴더

**HTML 요구사항:**

- 배경: 흰색(#ffffff) 또는 아주 옅은 그레이(#fafafa)
- 각 아이콘 카드 배경: 연한 회색 (흰 배경과 구분되며, 흰 계열 아이콘도 보이도록)
- 아이콘 프레임: 220×220px, `border-radius: 50px` (iOS 26 squircle 근사), 약한 drop shadow
- 아이콘은 inline SVG로 직접 구성 (외부 이미지 X)
- 각 카드 하단에 60·40·28px 미니 프리뷰 3개 (작은 사이즈 검증용)
- 각 카드에 컨셉명, 한 줄 설명, 네가 2단계에서 쓴 논리 + 트레이드오프 요약 표시
- 헤더에 추출한 팔레트(swatch + hex)와 앱 무드 한 줄 요약 표시

**스타일 원칙:**
- 불필요한 그라데이션·남용된 그림자·장식 요소 금지
- 타이포가 들어간다면 앱의 폰트 성향과 일치시킬 것
- 5개 아이콘은 **시리즈로 비교 가능**하도록 배경·accent 적용 방식을 일관되게

### 4단계: 네 추천 순위

HTML 하단 또는 응답 말미에, 5개 후보를 **네가 시니어 디자이너로서 추천하는 순위**로 매겨줘.
순위 근거를 한 줄씩 덧붙일 것.

그리고 픽이 결정되면 1024×1024 PNG 및 iOS 아이콘셋 전체 사이즈로 내보낼 수 있음을 안내.

## 제약

- 파일은 `devetym_icon_candidates.html` 1개만 생성. 중간 산출물 남기지 말 것.
- 외부 폰트/CDN/이미지 의존 X. 시스템 폰트와 inline SVG만.
- HTML은 단독으로 완전히 렌더되어야 함.
- 색상 추출이 애매하면 추측하지 말고 질문.
- 너의 판단을 나에게 양보하지 마라. "혹시 이런 방향도 좋으실까요?" 같은 수동적 톤 X. 전문가의 자신감으로 제시할 것.

## 출력 순서

1. 추출한 색상 팔레트 (hex)
2. 앱 무드 한 줄
3. 5개 후보 방향의 논리 (HTML에 들어가기 전 텍스트로 먼저 설명)
4. HTML 파일 생성 안내
5. 네 추천 순위 + 근거
