Italic e · period

사전 항목의 어원 표기 관습(이탤릭 표제어 + 마침표)을 그대로 레터마크로.

핵심 시각 요소
거대한 이탤릭 serif "e" + 우하단 olive 마침표 한 점.
왜 이 브리프에 맞는가
사전·언어학 책에서 이탤릭은 어원·외래어를 표기하는 활자 관습이다. period는 항목의 끝. 이 두 형태가 만나는 순간 뇌는 "사전 항목"으로 분류한다. 0.5초 카테고리 인식 측면에서 가장 강한 후보.
리스크
너무 절제되어 다른 사전류·읽기 앱과 차별화가 약하다. "e가 무엇의 e인지" 정보 없음.
28px 실루엣 앵커
e의 닫힌 카운터 형태와 우하단 olive 점 — 둘 다 작아져도 살아남는 단순 도형.

Open Book · cursor

펼친 책 실루엣(사전) 위에 깜빡이는 olive 터미널 커서(개발자).

핵심 시각 요소
중앙 fold가 있는 펼친 책 + 우측 페이지 위 olive 막대 커서.
왜 이 브리프에 맞는가
"책 = 사전"은 시장에서 가장 보편적·즉시적 시그널. 거기에 막대 커서를 얹으면 "개발자용 사전"이라는 두 번째 정보가 0.7초 시점에 따라온다. 메타포 충돌 없이 두 신호가 누적된다.
리스크
책 아이콘 시장 포화. 커서 없으면 그냥 "또 다른 책 앱". 차별화가 단일 막대에 의존하므로 28px에서 커서가 시각적으로 약해지면 평범해질 위험.
28px 실루엣 앵커
중앙이 V자로 꺾이는 펼친 책의 외곽선 — 가장 안정적인 형태 인식.

// Comment

olive 풀배경 + 흰 코드 코멘트 슬래시 + 사전 텍스트 라인.

핵심 시각 요소
상단의 큰 흰 `//` 두 줄 + 하단의 사전 페이지를 암시하는 짧은 흰 라인 두 줄.
왜 이 브리프에 맞는가
홈스크린 흑백·중성색 아이콘 사이에서 olive 풀배경은 강하게 튄다. `//`는 거의 모든 언어에서 "코멘트 = 설명/주석"이며, 그 아래 텍스트 라인이 "이 설명은 사전 항목 형태"임을 보강한다. 차별화 + 카테고리 인식을 동시에.
리스크
비개발자에겐 `//`가 빗금일 뿐. 단, 타겟이 개발자라 무관. 풀컬러 아이콘은 호불호가 강함.
28px 실루엣 앵커
olive 컬러 자체가 첫 번째 식별. 흰 평행 두 줄이 두 번째.

한글 ‘어’ · lime

다크 배경 위 라임 한글 ‘어’ — 한국어 개발자 정조준 레터마크.

핵심 시각 요소
오프블랙 풀배경에 라임 색 한글 ‘어’ 한 글자.
왜 이 브리프에 맞는가
한국 사용자에겐 ‘어’가 즉시 "어원 / 어휘 / 한국어 사전"을 발사한다. 영문 일색의 개발자 도구 아이콘 사이에서 한글 한 글자는 강력한 차별화이자 "이 앱은 내 모국어로 쓰여 있다"는 즉각적 정체성 신호.
리스크
한국어를 모르면 의미 불명. 글로벌 확장성 0. 미래 다국어 출시 시 재디자인 필수. MVP 한국 한정 전제 위에서만 정당화됨.
28px 실루엣 앵커
‘어’의 한글 구조(ㅇ + ㅓ) — 라임 컬러와 결합되어 작아져도 한글로 인지됨.

{ e } · braces

코드 중괄호(개발자) 안에 serif 이탤릭 e(어원) — 두 정체성의 정확한 교집합.

핵심 시각 요소
좌우 모노스페이스 중괄호 `{ }` + 가운데 olive serif 이탤릭 ‘e’.
왜 이 브리프에 맞는가
중괄호는 모든 개발자가 즉시 "코드"로 인식한다. 가운데 serif e는 사전·문헌의 활자. 두 신호가 한 글리프 안에서 융합되어 짧은 해시태그처럼 기억에 남는다. 중괄호의 두 곡선이 28px에서도 안정적인 외곽 실루엣을 만들어 식별 안정성도 높다.
리스크
중괄호 안에 글자를 넣는 표현은 개발자 도구류에서 흔한 관용 표현. 차별화는 'serif e'의 활자 선택에 의존하므로 거기를 부드럽게 처리해야 함.
28px 실루엣 앵커
좌우 두 곡선의 대칭 + 가운데 olive 색 한 점 — 색-형태 이중 앵커.

추천 순위

  1. { e } · braces card 05
    카테고리 인식(serif e) + 차별화(중괄호) + 28px 생존성(좌우 곡선) 셋이 균형. App Store 아이콘 그리드에서 마크처럼 기능. MVP 단계에서 가장 적은 리스크로 가장 큰 인식 효과.
  2. Italic e · period card 01
    사전 카테고리 인식만 보면 최강. 차별화는 약하지만 "처음 1주일 무한 안전" 픽. 추후 와이드마크 / 워드마크와 결합하기에도 가장 유연한 베이스.
  3. // Comment card 03
    홈스크린에서 튀는 능력 1위. 단점은 사전 시그널보다 "코드 코멘트"가 먼저 읽힐 수 있다는 것. 차별화·발견성을 카테고리 인식보다 우선한다면 1순위로 올림.
  4. 한글 ‘어’ · lime card 04
    한국 시장 정조준의 강력함. 한국어 사용자만 보면 즉시 "어원/어휘"로 분류된다. 글로벌 확장 계획이 없다면 2순위까지 올라갈 수 있음. 현재 글로벌 가능성을 고려해 4순위.
  5. Open Book · cursor card 02
    메타포 자체는 깨끗하지만 책 아이콘 시장이 너무 포화. 커서가 28px에서 약해지는 순간 평범한 "또 다른 책 앱"이 됨. MVP 차별화에 가장 약한 후보.
다음 단계 안내. 위 5개 중 하나를 픽하면 1024×1024 PNG와 iOS 아이콘셋 전체 사이즈 (20·29·40·58·60·76·80·87·120·152·167·180·1024 pt) 를 그대로 내보낼 수 있다. SVG 정의가 viewBox 1024×1024로 통일되어 있어 다른 사이즈도 무손실 스케일링 가능. 필요 시 라이트·다크 듀얼 톤 버전(특히 Card 01 / 05)도 같이 출력 가능.
